# 项目业务信息插槽

> 本文件供模板使用者在 clone 后补充自己的业务背景。
> 通常只需要修改本文件，不建议为了补充业务介绍而修改其他 `.ruler/*.md` 模板通用规则。

## 项目名称

Flutter Starter App

## 项目定位

本项目默认是面向中大型 Flutter 应用的快速开发模板。使用者可在这里替换为实际业务定位，例如：

- 电商 App
- 内部管理 App
- 内容社区 App
- IoT 控制 App
- 企业工具 App

## 主要业务模块

请按实际项目补充模块清单。示例：

- 用户认证
- 首页
- Todo 示例
- 个人中心

## 业务术语

| 术语 | 含义 |
|:---|:---|
| 示例：租户 | 当前项目中的组织或客户隔离单位 |
| 示例：设备 | 用户绑定或管理的硬件终端 |

## 特殊业务约束

请补充当前项目区别于模板默认规则的业务约束。示例：

- 必须兼容 Android 8+。
- 登录态需要支持多租户。
- 接口错误码由后端统一定义。
- 所有用户可见文案必须接入国际化。

## AI 协作偏好

请补充当前项目希望 AI 优先遵守的协作偏好。示例：

- 新增 Feature 时优先补充 ViewModel 和 Repository 测试。
- 新增页面文案默认使用中文，并同步补充国际化资源。
- 业务页面导航优先使用 `package:flutter_starter_app/header.dart` 中统一导出的 route class 与 `appRouterProvider`。

## 与模板规则的关系

- 本文件用于补充业务背景，不替代模板通用架构规则。
- 若本文件内容与模板规则冲突，AI 应先指出冲突，再确认是否需要调整 `.ruler/` 中的模板通用规则。
- 不要在本文件记录密钥、Token、账号密码、生产环境地址等敏感信息。
